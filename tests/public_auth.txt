## My new hypothesis

Look back at image 1 of `auth.py` line 92:

```python
_bearer_scheme = HTTPBearer(auto_error=True)
```

And then line 100–104 — `_get_keycloak_user` declares `credentials: HTTPAuthorizationCredentials = Depends(_bearer_scheme)`.

`HTTPBearer(auto_error=True)` is a FastAPI security scheme. **When a route depends on it (directly or indirectly), and no Bearer token is present, FastAPI auto-rejects with 403 "Not authenticated" before the handler runs.**

So somewhere, our `/ready-for-export` endpoint must be transitively depending on either `_bearer_scheme` or `_get_keycloak_user`. The question is where.

Three places it could be entering:
1. The router declaration in `project_routes.py` has `dependencies=[Depends(...)]` — but earlier screenshots showed it doesn't
2. Something in `get_session` is pulling auth in — but we saw it doesn't
3. **The router is included in `api.py` with a global dependency** — possible but I didn't see this in the api.py screenshot
4. **There's something else in `core/auth.py` past what I've seen** — possible, image 2 ends at line 177 but the file might continue
