```python
def _ensure_status_columns(eng):
    """Add status and submitted_at columns to project table if they don't exist.
    These power the submission lifecycle (draft -> submitted -> exported)
    consumed by downstream enrichment / Archer publisher services.
    """
    with eng.connect() as conn:
        # status
        result = conn.execute(text("""
            SELECT column_name FROM information_schema.columns
            WHERE table_name = 'project' AND column_name = 'status'
        """))
        if not result.fetchone():
            log.info("Adding status column to project table...")
            conn.execute(text(
                "ALTER TABLE project ADD COLUMN status VARCHAR(20) NOT NULL DEFAULT 'draft'"
            ))
            conn.commit()
            log.info("status column added (default 'draft' for existing rows)")

        # submitted_at
        result = conn.execute(text("""
            SELECT column_name FROM information_schema.columns
            WHERE table_name = 'project' AND column_name = 'submitted_at'
        """))
        if not result.fetchone():
            log.info("Adding submitted_at column to project table...")
            conn.execute(text(
                "ALTER TABLE project ADD COLUMN submitted_at TIMESTAMP NULL"
            ))
            conn.commit()
            log.info("submitted_at column added")

        # index on status for the /ready-for-export query path
        result = conn.execute(text("""
            SELECT indexname FROM pg_indexes
            WHERE tablename = 'project' AND indexname = 'idx_project_status'
        """))
        if not result.fetchone():
            log.info("Creating idx_project_status index...")
            conn.execute(text(
                "CREATE INDEX idx_project_status ON project (status)"
            ))
            conn.commit()
            log.info("idx_project_status index created")


# ---------------------------------------------------------
# Seed data
# ---------------------------------------------------------
```

## What this does

- **Idempotent** — uses the same `information_schema.columns` check pattern that `_ensure_owner_email` and `_ensure_archer_columns` use. Restarting the backend after the migration runs is safe; it sees the columns already exist and skips.
- **Index check uses `pg_indexes`** instead of `information_schema.columns` because indexes aren't columns — that's the correct Postgres pattern.
- **Default value `'draft'`** on the `status` column means every existing project gets `status='draft'` automatically when the migration runs. No data loss, no manual UPDATE needed.
- **Logs at every step** so when the backend starts up after the migration, you'll see clear log lines like `Adding status column to project table...` and `status column added (default 'draft' for existing rows)`. After the first successful run, subsequent restarts log nothing for this function (since all the checks return existing rows).
